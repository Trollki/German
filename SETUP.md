# SETUP — Быстрая настройка Firebase (на русском)

Ниже пошаговая инструкция, чтобы приложение начало работать и все данные сохранялись в облаке (Firestore) на всех устройствах.

1) Проверить, что конфигурация в index.html уже вставлена
- Откройте `index.html` в корне репозитория — вверху есть объект `firebaseConfig`.
- Если он содержит реальные значения (apiKey, projectId и т.д.), оставьте как есть.

2) Создать пользователя для входа (Email/Password)
- Откройте Firebase Console: https://console.firebase.google.com
- Выберите проект (projectId указан в `firebaseConfig`, например `german-9e4d6`).
- Слева: Authentication → Sign-in method → включить Email/Password.
- Затем: Users → Add user:
  - Email: `yulia@german.de`
  - Password: `Ger20`

3) Включить Firestore и поставить правила безопасности (Production)
- В Firebase Console: Firestore Database → Create database.
- Выберите режим: "Production (с защитными правилами)".
- Когда откроется редактор правил, замените их на эти (вставьте и нажмите Publish):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

Эти правила гарантируют, что каждый пользователь может читать и писать только в своей ветке `/users/{uid}`.

4) Проверка `storageBucket` (опционально)
- В `firebaseConfig` в `index.html` поле `storageBucket` должно совпадать с тем, что указано в Project Settings → General → Your apps.
- Обычно это `PROJECT_ID.appspot.com`.

5) Как запустить и проверить работу
- Откройте `index.html` в браузере (двойной клик или через локальный сервер).
- На странице входа используйте email `yulia@german.de` и пароль `Ger20`.
- После входа попробуйте: добавить ученика → добавить урок → сохранить заметки.
- Откройте страницу на другом устройстве (другой компьютер/телефон) и войдите тем же аккаунтом — данные должны появиться автоматически.

6) Что делать, если что-то не работает
- Ошибка "Неверный пароль": проверьте, что пользователь создан в Firebase Authentication и что вы ввели правильный пароль `Ger20`.
- Пустой экран/вечная загрузка: откройте DevTools (F12) → Console, сообщите текст ошибки.
- Нет записи в Firestore: убедитесь, что правила опубликованы и Auth выполнен успешно.

7) Безопасность
- Пароль `Ger20` временный — когда всё настроено, смените пароль на более надёжный в Firebase Console → Authentication → Users → Edit.
- Никогда не публикуйте этот репозиторий с реальными приватными ключами в публичных местах, если вы не хотите, чтобы кто-то видел ваш apiKey.
